# TCP/IP
TCP(transmission control protocol)是可靠的，面向连接的协议，通过三次握手建立连接、四次挥手关闭连接，并在数据传输过程中，使用确认、重传、流量控制、拥塞控制等机制来保证数据的可靠传输。
## TCP/IP 五层模型
1. 应用层：计算机 ｜ HTTP、HTTPS、SMB、FTP、DNS、ROP、SSH、VNC、SOCKS
2. 传输层：防火墙（过滤端口）｜TCP、UDP
3. 网络层：路由器、三层交换机 ｜ IPV4、IPV6、ICMP、ARP
4. 数据链路层：交换机（二层）｜ 以太网
5. 物理层：网线（双胶线）、光纤、网卡（网络适配器）
   * 二层交换机：基于MAC地址在同一VLAN中转发数据帧，不能跨网通信*
   * 三层交换机：既能像二层交换机一样转发帧，又能基于IP地址在不同VLAN或子网间路由器数据包，实现跨网高速通信*
   * VLAN*：虚拟局域网，用于在物理网络上隔离广播域；
   * WLAN*：无线局域网，通过无线Wi-Fi技术实现设备接入，二者结合使用。
   * 物理网络*：由实际硬件设备和传输介质构成的网络基础设施![1764832011916](image/计算机网络知识/1764832011916.png)
   * 广播域是指广播帧能够达到的所有设备合集*![1764832124194](image/计算机网络知识/1764832124194.png)
* ### 封装&解封装
  * 发送方在从上层至下层传递数据时，每经过一层就添加该层的头部（也可能是尾部）的信息，该过程为封装
  * 接收方在从下层至上层接收数据时，每经过一层就剥离该层的数据，该过程为解封装

### TCP的特性：
1. 面向连接：在数据传输前需要建立连接，在数据传输结束后需要关闭连接
2. 可靠性：通过确认机制和重传机制，确保数据不重复不丢失
3. 有序性：数据是按照发送顺序到达接收方
4. 流量控制：通过滑动窗口机制，动态调整发送频率
5. 拥塞机制：通过慢启动、拥塞避免等算法，避免网络堵塞
   *注：窗口滑动机制是用于实现流量控制的机制，它允许发送方在没收到ACK包前连续发送多个数据段，从未提升带宽的利用率；同时依据接收方通告的窗口大小动态调整发送速率，避免接收方缓冲区溢出。*

### 三次握手

1. 客户端发送一个SYN包同步请求给服务器，表示请求建立连接
2. 服务器接收SYN包后，回复一个SYN-ACK的包，表示同意建立连接
3. 客户端收到SYN-ACK后，再发一个ACK的包，表示连接已经建立
   *注：SYN （synchronize）、ACK（acknowledgment）*
   ![1764659519561](image/计算机网络知识/1764659519561.png)

### 四次挥手

1. 客户端发送FIN包给服务器，请求关闭连接
2. 服务器收到FIN包后，发送ACK包表示确认
3. 服务器再发送FIN包，表示服务器准备关闭连接
4. 客户端收到FIN包后，再发送ACK包，表示确认。连接正式关闭
